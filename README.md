# SpreadScanner — фаза 0: событийный сканер кросс-биржевых спредов перпов

Один процесс: WS bookTicker со всех бирж → нормализованные котировки в памяти →
инкрементальный детектор спредов с гистерезисом → три потока данных:

- `data/opportunities.db` (SQLite) — журнал событий: пара бирж, длительность,
  максимум валового/чистого (после 4 тейкер-комиссий) спреда, глубина у вершины;
- `data/snap_*.parquet` — секундные снапшоты всех котировок (исследования);
- `data/event_ticks_*.parquet` — полная детализация котировок внутри событий;
- `data/funding_*.parquet` — фандинги раз в минуту.

## Запуск

    python scanner.py --minutes 0 --max-symbols 0 --exchanges gate,okx,mexc,bitget

`--minutes 0` = бесконечно. Ctrl+C — буферы дописываются.

## Биржи

| Биржа | Канал | Статус |
|---|---|---|
| gate | futures.book_ticker | ok, ~20мс |
| okx | bbo-tbt (размер × ctVal) | ok, ~40-60мс |
| bitget | books1 | ok, ~30мс |
| mexc | push.ticker (per-symbol) | МЕДЛЕННЫЙ: ~1 сообщение/сек/символ, лаг 1-2.5с. Спреды с ногой mexc до апгрейда на sub.depth считать фантомами |
| binance | bookTicker (combined) | гео-блок 451 с локальной машины; работает с VPS |
| bybit | tickers merge | гео-блок 403 CloudFront локально; работает с VPS |

Сопоставление тикеров между биржами — только точное совпадение строки
(1000PEPEUSDT != PEPEUSDT: защита от кратных контрактов; кратные маппинги — TODO).

## Пороги

Событие открывается при валовом спреде > 0.30%, закрывается < 0.10% (гистерезис).
Чистый спред = валовый − 2×(taker_buy + taker_sell). Комиссии — константы в
коннекторах (VIP-уровни/рибейты уточнят).

## Извлечённые уроки смоук-теста (2026-07-18, 5 минут, 40 символов)

1. Медленная нога рождает фантомы: «+4-6%» спреды с участием mexc — артефакт
   его секундного тикер-канала. Качество данных = качество сканера.
2. Реальные события (gate↔bitget, оба фида быстрые): +0.15..0.79% чистыми,
   живут 0.1-10с, ёмкость $100-1400 у вершины стакана (top-of-book).
3. «Жирный и долгий» спред = нет ёмкости: 4USDT +0.26% на 109 секунд — $5 сверху.
